# Пояснения по лабораторной работе №1

Эта лабораторная работа является вводной и учит простой работе с вводом/выводом в С.
Вот то, про что нужно почитать перед началом:
1. Функция для считывания scanf()
2. Функция для печати printf()
3. Для многих вариантов — функции из модуля <math.h>

```c
#include <stdio.h> // Подключаем основную библиотеку
#include <stdio.h> // Подключаем основную библиотеку

int main() { // Объявляем основную функцию
    SetConsoleOutputCP(CP_UTF8); // Устанавливаем кодировку UTF-8 для вывода русских символов
    int amount; // Объявляем переменную для хранения суммы в копейках
    int coins1 = 0, coins5 = 0, coins15 = 0; // Объявляем переменные для хранения количества монет

    printf("Введите сумму в копейках: "); // Выводим запрос на ввод суммы в копейках
    scanf("%d", &amount); // Считываем сумму в копейках

    coins15 = amount / 15; // Определяем, сколько монет достоинством 15 копейек потребуется
    amount = amount % 15; // Убираем использованные монеты

    coins5 = amount / 5; // Определяем, сколько монет достоинством 5 копеек потребуется
    amount = amount % 5; // Убираем использованные монеты

    coins1 = amount; // Определяем, сколько монет достоинством 1 копейку потребуется

    printf("Монет достоинством 1 копейка: %d\n", coins1); // Выводим количество монет достоинством 1 копейка
    printf("Монет достоинством 5 копеек: %d\n", coins5); // Выводим количество монет достоинством 5 копеек
    printf("Монет достоинством 15 копеек: %d\n", coins15); // Выводим количество монет достоинством 15 копеек

    return 0; // Завершаем программу с кодом 0
}
```
> [!IMPORTANT]
> Если Вы используете не Windows, а, например, macOS, то проблем с кодировкой быть не должно — в таком случае игнорируйте следующие строки:
> ```c
> ...
> #include <windows.h> // Подключение модуля windows.h для установки кодировки вывода
> ...
>      SetConsoleOutputCP(CP_UTF8); // Устанавливаем кодировку UTF-8 для вывода в консоли русских символов: иначе будут иероглифы
> ```
>
> В операционных системах семейства Linux могут быть свои заморочки с кодировкой. Если что-то работает не так, как ожидается, то заглядывайте в Google :)
